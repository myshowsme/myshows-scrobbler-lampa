# MyShows scrobbler plugin for Lampa

Плагин для [Lampa](https://github.com/yumata/lampa-source), который автоматически отправляет
прогресс просмотра в ваш профиль [MyShows](https://myshows.me): следит за плеером и передаёт
прогресс просмотра. Какая именно это серия или фильм — определяет бэкенд MyShows, плагин
шлёт только идентификаторы и прогресс, по умолчанию вы решаете добавить данный просмотр в 
профиль или нет. Весь прогресс отправленный из плагина отображается в разделе "История 
просмотров" на сайте MyShows.

## Возможности

- Автоматический скробблинг — ничего не нужно искать вручную, все собрано в одной ленте.
- Сериалы и фильмы; смена серии подхватывается на лету.
- Настраиваемый порог «просмотрено» (50–95 %).
- Токен проверяется прямо при вводе — сразу видно, что связь есть.
- Устойчивость к обрывам сети: прогресс досылается, сессия не теряется.
- Профили Lampa: у каждого профиля аккаунта свой токен и свой выключатель скробблинга.

## Установка

1. В Lampa: **Настройки → Расширения → Добавить плагин по URL**.
2. Вставьте ссылку:
   ```
   https://myshowsme.github.io/myshows-scrobbler-lampa/myshows.js
   ```
3. Откройте настройки плагина «MyShows», вставьте токен из профиля MyShows — он проверяется
   автоматически при вводе (тост с результатом).

## Настройки

| Параметр            | Назначение                                               | Дефолт     |
| ------------------- | -------------------------------------------------------- | ---------- |
| Скробблинг          | вкл/выкл отправки                                        | вкл        |
| Токен MyShows       | Bearer-токен из профиля; проверяется при вводе (`/check`)| —          |
| Порог «просмотрено» | % для засчёта просмотра (50/60/70/80/90/95)              | 80 %       |
| Адрес API           | базовый URL scrobble API (тест/self-host)               | myshows.me |

Все настройки хранятся на каждый профиль аккаунта Lampa отдельно: при переключении профиля
подхватываются его значения. Новый профиль начинает с настроек по умолчанию (пустой токен) —
ничего не наследуется от других профилей. Исключение — обновление со старой версии плагина:
первый профиль подхватывает уже введённые настройки устройства (токен не «теряется»).

## Как засчитывается просмотр

- Просмотр засчитывается, когда вы досмотрели до выбранного порога (по умолчанию 80 %).
- Недосмотр ниже порога — не засчитывается.
- Если закрыть плеер до порога — ничего не отправляется.
- Переход на другую серию закрывает прежнюю сессию и открывает новую.

## Для разработчиков

### Сборка

Источник правды — **`myshows.full.js`** (правишь только его). Install-артефакт
**`myshows.js`** минифицируется из него и **руками не редактируется**:

```bash
pnpm install      # один раз
pnpm run build    # myshows.full.js -> myshows.js (terser, ecma 5)
```

Один самодостаточный файл-плагин (IIFE, чистый ES5, только через `window.Lampa`). Версия
пишется в консоль при загрузке, чтобы видеть живую сборку. Хостинг — GitHub Pages из корня
ветки `main`.

### Как работает (технически)

Жизненный цикл скроббла через scrobble REST API с Bearer-токеном:

- старт воспроизведения → `POST /start`;
- во время просмотра → `POST /pause` как heartbeat прогресса (раз ~30 с);
- достижение порога просмотра → `POST /stop` (один раз за сессию);
- закрытие плеера: `/stop` отправляется только если порог уже достигнут, иначе ничего не шлётся;
- смена серии (новая сигнатура `tmdb|season|episode`) закрывает старую сессию и открывает новую.

Устойчивость: транзиентные ошибки самоизлечиваются следующим heartbeat (интервал укорачивается
до ~10 с); после 3 ошибок подряд интервал замедляется ×4 (самовосстановление при успехе);
`401/403` гасит сессию и помечает токен невалидным.
